# Nome del prodotto/pacchetto: MuleSoft

## Breve descrizione sintetica
Integration and API platform for connecting Salesforce, external systems, events, APIs, data and process orchestration.

## Oggetti principali
- APIs, RAML/OAS specifications, Mule apps, connectors, policies.
- Salesforce connected apps, named credentials, external credentials.
- Platform events, CDC, integration users, external IDs.

## Funzionalita principali
- API-led connectivity and orchestration.
- System/process/experience APIs.
- Connectors, transformations, governance and monitoring.

## Configurazioni principali
- API Manager, Anypoint Exchange, Runtime Manager, environments.
- Authentication, connected apps, certificates and secrets.
- Salesforce connectors, bulk API, event subscriptions.
- Error handling, retry policy and observability.

## Best practice
- Define API ownership, contracts and versioning before implementation.
- Use named credentials and integration users with least privilege.
- Plan idempotency, retries, limits and error queues.
- Avoid point-to-point integrations when reusable APIs fit.

## Contesto versione
Prima della pianificazione leggere `references/salesforce-current-version.md`. Verificare la release della target org e la disponibilita' effettiva delle funzionalita'; per prodotti distribuiti anche come managed package o add-on, verificare la versione installata nella org quando rilevante.
